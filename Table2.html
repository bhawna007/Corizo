JavaScript: Difference Let Var Const

No. let var Const
Scope let is block-scoped, meaning it is only accessible within the block {} in which it is declared. var is function-scoped, meaning it is accessible within the function where it is declared. If declared outside a function, it becomes globally scoped. Like let, const is block-scoped.
Hoisting let is also hoisted, but unlike var, it is not initialized with undefined. Accessing it before declaration results in a ReferenceError due to the temporal dead zone (TDZ). Variables declared with var are hoisted to the top of their scope, meaning the declaration is moved to the top during the compile phase, but not the initialization. const behaves similarly to let in terms of hoisting and the temporal dead zone.
Reassignment and Redeclaration You can reassign a let variable, but you cannot redeclare it in the same scope. You can reassign and redeclare a var variable within the same scope. const is used to declare variables that cannot be reassigned. The variable itself cannot be reassigned, but if it is an object or array, the contents can be modified.
Example 
function example() {
                console.log(y); // ReferenceError: 
                Cannot access 'y' before initialization
                let y = 20;
                console.log(y); // 20
            }
            example();
 
function example() {
                console.log(x); // undefined 
                (due to hoisting)
                var x = 10;
                console.log(x); // 10
            }
            example();
