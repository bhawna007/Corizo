Unit Definition Usage Pros Cons Example
px(Pixels) Pixels are absolute units. One pixel (px) is a fixed size on the screen, and it doesn't change regardless of other elements or screen size. Use px when you need precise control over the size of an element, such as borders, images, or specific text sizes. Consistent across different browsers and devices. Less flexible, not responsive, and doesn't scale with user settings. p { font-size: 16px; }
Percentage (%) A percentage is a relative unit that defines size relative to the parent element's size. Often used for fluid layouts, width settings, or when defining sizes relative to other elements. Responsive and flexible, as the size adjusts based on the parent element. Can be tricky to manage with nested elements and may lead to unexpected results if not carefully handled. div { width: 50%; }
Root em (rem) rem stands for "root em". It is relative to the root element () font size. If the root font size is 16px, then 1rem equals 16px. Use rem when you want to ensure consistency across the entire document and allow easy scaling based on the root font size. Consistent scaling across the entire page, making it easier to adjust the entire document by changing the root font size. Requires knowledge of the root font size for accurate calculations. p { font-size: 1.5rem; /* 24px if root font size is 16px */ }
Em (em) em is a relative unit, but it is relative to the font size of its parent element. If the parent font size is 16px, 1em equals 16px. Useful when you want to size an element relative to its parent or for creating scalable components. Flexible and can adapt to different contexts, making it great for scalable layouts. Can be confusing when deeply nested, as the value of em compounds based on parent elements. div { font-size: 2em; /* 32px if the parent font size is 16px */ }
